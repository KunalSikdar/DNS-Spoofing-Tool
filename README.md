This project demonstrates the concept of DNS spoofing, also known as DNS cache poisoning, where an attacker injects falsified information into DNS responses to redirect traffic from a legitimate domain to a different IP address. As a result, users attempting to access a trusted website may unknowingly be redirected to a malicious server controlled by an attacker.

The tool simulates how DNS responses can be intercepted and modified within a controlled environment, illustrating the risks associated with insecure DNS communication. By manipulating DNS resolution, the project highlights how attackers can alter the normal domain-to-IP mapping process and influence user traffic.

This project is intended for educational and cybersecurity research purposes, helping learners understand the mechanics behind DNS spoofing, network vulnerabilities, and the importance of secure DNS practices.
